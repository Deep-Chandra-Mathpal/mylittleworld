## System Requirements

### Minimum Hardware
```
- CPU: Intel i5 equivalent or better
- RAM: 16GB (32GB recommended for labs)
- Storage: 500GB SSD (fast performance)
- Network: Ethernet for stable connection
```

### Operating System
```
Recommended:
- Linux (Ubuntu 20.04 LTS or later)
- Kali Linux 2024.x
- Debian 12

Compatible:
- macOS 12+
- Windows 11 (WSL2 recommended)
```

### Virtual Machine Requirements
```
For lab environment:
- Hypervisor: VirtualBox 7.0+, VMware Player, Hyper-V
- Per VM Allocation:
  - Attacker (Kali): 4 CPU, 8GB RAM, 40GB storage
  - Target (WebGoat): 2 CPU, 4GB RAM, 20GB storage
  - Target (Metasploitable): 1 CPU, 2GB RAM, 20GB storage
  - Target (Custom): 2 CPU, 4GB RAM, 20GB storage
```

## Anti-Virus/Firewall Configuration for Testing

```bash
# Disable protections (for lab only!)

# Windows Defender
# Settings > Virus & threat protection > Manage settings
# Disable: Real-time protection

# Ubuntu AppArmor
sudo systemctl stop apparmor
sudo systemctl disable apparmor

# UFW Firewall
sudo ufw disable

# SELinux
sudo setenforce 0
```

**WARNING**: These configurations are lab-only. Never use in production!

## Troubleshooting Installation

### Python Module Not Found
```bash
pip install --upgrade [module_name]
pip install --force-reinstall [module_name]
```

### Permission Denied (Scapy, raw sockets)
```bash
sudo setcap cap_net_admin,cap_net_raw+ep $(python3 -m site --user-site)/scapy/arch/linux.py

# Or run with sudo
sudo python3 exploit.py
```

### GDB Permission Issues
```bash
echo 'kernel.ptrace_scope = 0' | sudo tee -a /etc/sysctl.conf
sudo sysctl -p
```

### Kali Linux Specific
```bash
# All tools pre-installed
sudo apt update
sudo apt full-upgrade
```

## Security Considerations

### Lab Isolation
```
- Lab network completely isolated from production
- No internet access for target machines
- Host firewall enabled
- Host antivirus running
- Regular backups of lab
```

### Safe Practices
```
- Never test on systems you don't own
- Always get written authorization
- Work in isolated lab environment
- Use snapshots before testing
- Document all activities
- Clean up after testing
```

## Performance Optimization

### For Slow Systems

```bash
# Reduce Docker memory

# Instead of full WebGoat, use lightweight DVWA

```
